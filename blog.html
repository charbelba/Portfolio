How to create a Java Spring Boot RestController for CRUD operations
21th October 2023

In this example, we are working with a simple in-memory list of tasks for demonstration purposes. In a real-world scenario, you would typically interact with a database or some other data storage mechanism to perform these CRUD operations on tasks. The controller handles HTTP requests and responses for creating, retrieving, updating, and deleting tasks. It's a basic implementation to illustrate how these CRUD operations work in a Spring Boot application. In this example, we've defined a TaskController with four endpoints:

  • POST /tasks: Create a new task.
  • GET /tasks/{taskId}: Retrieve a task by its ID.
  • PUT /tasks/{taskId}: Update an existing task by its ID.
  • DELETE /tasks/{taskId}: Delete a task by its ID.
Creating our RestController Class

We need to create first our controller using @RestController annotation it acts as an entry point to our methods and we also need to give it a path using @RestMapping("/task") suppose we are working locally we can access this controller via http://127.0.0.1:8080/task

Updating a Task

This function allows you to update an existing task by its ID. It takes the taskId as a path variable and the updated task details in the request body as a JSON object. It searches for the task with the specified ID and, if found, updates its name and description with the values provided in the request. If the task does not exist, it throws a TaskNotFoundException.
